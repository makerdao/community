# Brand Guide

The following are basic visual guidelines for all Comm-Dev branded content, including UI, marketing materials, and social media assets.

Here you will find:

- Color

- Illustrations
- Document templates

## Color

There are two variations of our color palette: light mode and dark mode. Every color is named semantically, or after its purpose as opposed to its appearance (e.g. "error" instead of "red"). Semantic color names allow this site to seamlessly toggle between light and dark modes. (Haven't tried? Click the sun and moon icons in the site navigation.)

In a UI context, the suffixes `Emphasis`, `Alt`, and `Muted` are generally used for interactive elements to differentiate interaction states or provide increased accessibility. In an illustration or editorial context, they can be used more flexibly at the artist or designer's discretion.

Additional notes:

- For marketing and editorial materials (in other words, anything that doesn't have two "modes" of appearance), default to the "light mode" palette.
- If a color only has one hex code listed, it is the same in both light and dark modes.

## Typography

### Fonts

We use the font **FT Base** on this site, which requires a font license to be used publicly. For internal prototypes, sketches, or other proofs-of-concept that aren't public, FT Base offers a [trial font](https://www.frostype.com/base) that anyone can download from their foundry site and use.

When a free font is needed (e.g. in a PowerPoint template), we use [Work Sans](https://fonts.google.com/specimen/Work+Sans).

### Scale

Our type scale and spacing systems are derived from [Dai UI.](https://dai-ui.now.sh/)

## Illustrations

Community contributor [Richard Rosa](https://chat.makerdao.com/direct/RichardR) created all the illustrations on this site, including a collection of spot illustrations.

### Using and Creating Illustrations

Consider incorporating an illustration if:

- You are trying to break up a dense amount of information
- You are asking something of a user (e.g. asking a user to take time to complete a feedback survey)
- A user may be frustrated or confused

If you want to commission or create new illustrations for this site, keep the following considerations in mind:

- All illustrations should use our brand palette
- All illustrations should be usable on both light and dark backgrounds
- All illustrations should be saved as .svg or .png

### Spot Illustrations

These illustrations can be used alone, or combined and remixed to create a whole scene. They can be found in `static/images/illustrations`.
